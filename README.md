# claw-link signaling server

WebSocket signaling server for [claw-link](https://github.com/TITOCHAN2023/ClawLink) WebRTC peer coordination.

## Room flow

```
Peer A connects to /             → server generates roomId → returns in ready
Peer A shares roomId out-of-band
Peer B connects to /<roomId>     → server matches → P2P handshake begins
```

Room ID is a 32-char hex string (128-bit) generated by the server. **It is both the room address and the authentication token** — only peers who know the ID can join. Treat it like a password.

> **⚠️ Keep your Room ID secret.** Anyone with the Room ID can join the room and communicate with the peer inside. Never post it publicly, never commit it to version control, and only share it through secure private channels (encrypted DM, face-to-face, etc.).

## Quick Start

```bash
pip install -r requirements.txt
python server.py
```

Default: `ws://0.0.0.0:8765`

---

## Configuration (env vars)

| Variable | Default | Description |
|----------|---------|-------------|
| `SIGNAL_HOST` | `0.0.0.0` | Bind address |
| `SIGNAL_PORT` | `8765` | Listen port |

| `SIGNAL_TRUSTED_PROXIES` | `127.0.0.1,::1` | Trusted reverse proxy IPs (comma-separated) |

Built-in limits: rate 30 msg/s per IP, IP cooldown 3s, max 100 rooms, 6 conns/IP, 5 rooms/IP, empty room TTL 60s, hard TTL 2h.

## Deploy

```bash
cp server.py requirements.txt /opt/claw-link-signal/
cp claw-link-signal.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now claw-link-signal
```

### Nginx (WSS)

```nginx
location /signal/ {
    proxy_pass http://127.0.0.1:8765/;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_read_timeout 86400;
}
```

Trailing `/` in `proxy_pass` strips the `/signal/` prefix.

## Protocol

| Message | Direction | Payload |
|---------|-----------|---------|
| `ready` | server -> client | `{role, roomId, message}` |
| `peer-joined` | server -> client | _(none)_ |
| `offer` | client -> client | raw SDP string |
| `answer` | client -> client | raw SDP string |
| `ice` | client -> client | `{candidate, sdpMid, sdpMLineIndex}` |
| `peer-left` | server -> client | _(none)_ |
| `error` | server -> client | error string |
